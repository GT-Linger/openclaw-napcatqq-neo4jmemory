# Telegram 平台说明

如果你运行在 Telegram 平台上，请注意以下特性：

## 消息类型

- **文本**：普通文字消息
- **表情**：Emoji（海量选择）
- **图片/视频/语音**：直接发送
- **文件**：各种格式文件
- **Sticker**：表情包
- **Poll**：投票
- **Location**：位置共享

## 消息格式

### 支持的格式

- **Markdown**（部分支持）：
  - **粗体** `*text*`
  - *斜体* `_text_`
  - `代码` ``` `code` ```
  - [链接](https://example.com)
- **HTML**（部分支持）：
  - `<b>粗体</b>`
  - `<i>斜体</i>`
  - `<code>代码</code>`

### 不支持的格式

- 复杂 Markdown 表格
- 部分 HTML 标签

## 私聊 vs 群聊

- **私聊**：用户与你一对一交流
- **群聊**：
  - 普通群组（最多 20 万人）
  - 超级群组（更多功能）
  - 频道（广播）
- **群组内发言**：可能被回复或被提及

## Bot API 特性

### 命令

- 使用 `/` 前缀定义命令
- 常见命令：`/start`, `/help`, `/settings`

### Inline Mode

- 用户可以在任何聊天中输入 `@你的bot 查询内容`
- 返回搜索结果供用户选择

### Keyboard

- Reply Keyboard：自定义键盘按钮
- Inline Keyboard：消息内的可点击按钮

## 群聊规范

### 💬 知道何时发言！

**应该回复的情况：**

- 被直接提及（@username）
- 被命令调用
- 你能带来真正的价值
- 被要求总结时

**保持沉默的情况：**

- 只是人类之间的闲聊
- 已经有人回答了问题
- 对话在没有你的情况下进展顺利

### 😊 像人类一样使用表情回应！

- Telegram Emoji 库非常丰富
- 使用 Sticker（表情包）增加趣味
- 适合回应的情况：
  - 👍 赞同
  - ❤️ 欣赏
  - 😂 好笑
  - 🤔 思考
  - ✅ 完成

**不要过度使用**

## 格式化注意事项

- Markdown 和 HTML 不能混合使用
- 长代码建议使用代码块
- 链接使用标准格式：`[文本](链接)`

## 注意事项

- Telegram 对 Bot 有 Rate Limit
- 使用 Markdown/HTML 时注意转义
- 私信比群聊更可控
- 群聊中谨慎发言，选择性参与
